# Data

`responses.json` is generated by `1_generate_data.py`.

Run `python 1_generate_data.py` from the project root to regenerate it. Requires a Groq API key (see main README for setup).

## Format

Each entry:
```json
{
  "topic": "poetry",
  "prompt_index": 3,
  "prompt": "What makes a poem different from other forms of writing?",
  "token_lengths": [14, 8, 10, 5, 12, 9, 7, 3, ...]
}
```

## generation_log.txt

Captured terminal output from running `1_generate_data.py`. Shows the actual prompts sent, the chunk count returned per response, and the per-topic sanity check averages (poetry ~555 chunks, python_code ~616, medical ~621, cooking ~592, legal ~522). Kept as evidence of what the data collection run actually produced.

## example_pcap_output.txt

Sample output from running `extract_size_pcap.py` against a real packet capture of a Gemini conversation in the browser (Bonus 1 exploration). These are real packet sizes, not API chunk lengths — much larger and noisier, since one packet can carry several tokens or a UI/control message. Kept here as evidence the script works against real traffic; not yet integrated into the main classifier pipeline.
